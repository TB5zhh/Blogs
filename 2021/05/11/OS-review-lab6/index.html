文件接口

文件Trait

所有文件需要实现如何读出/写入。

stdin

限定用户缓冲区长度为1。循环从RustSBI获取输入的字符，如果未获取字符则放弃CPU资源并将进程放入进程池；如果获取了字符则退出循环并将字符写入缓冲区。返回读取的字符数（1）

通过UserBuffer获取字节写入位置？

stdout

将缓冲区中内容打印至控制台，返回实际打印的字符数量。

文件描述符表（文件引用表）

实现为Vec<Option<Arc<dyn File + Send + Sync>>>，文件描述符作为该数组的下标访问具体的文件引用。

使用Option用于指示文件描述符（指定的文件位置）是否空间，Some表示已被占用，None表示空闲。使用Arc获得共享引用能力，使用dyn关键字利用运行时多态、对抽象方法在运行时查找具体实现。

在新创建进程时（INITPROC），将stdin，stdout，stderr放入文件描述符表中；在fork时，子进程需要完全继承父进程的文件描述符表。

管道

基于文件的管道

通过pipe系统调用打开管道，将读端和写端作为文件保存在文件描述符表。

关闭管道端口时，使用Option::take()方法，将文件描述符指示的表中位置置为None即可。当文件的引用计数减为0时，文件占用的资源被自动回收。

管道端口被抽象为Pipe类型，维护Arc<Mutex<PipeRingBuffer>>类型的缓冲区，该缓冲区由读端与写端共享。

实现FileTrait。

  • 读取语义为从文件中读取应用缓冲区大小的字符。文件（循环队列）中不存在足够字符时暂停并换出进程，等待循环队列中有新的字符后再继续读取。
    • 实现：若管道为空，则检查是否有可用写端，若没有则返回0否则放弃CPU资源以等待写入段写入；若管道非空，将管道内容写入缓冲区，如果缓冲区写满则退出，否则继续循环。
  • 写入语义为向文件写入缓冲区中全部内容，文件（循环队列）中不存在可写入空间时暂停并换出进程，等待循环队列中有新的空间时再写入。
    • 实现：若管道为满，放弃CPU资源等待管道中新空闲位置产生；为管道非满，则将缓冲区内容写入文件，直到全部写入。

PipeRingBuffer可以创建新的读端/写端，提供读取和获取状态的方法。
